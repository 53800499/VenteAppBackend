Table 2 : usersGestion des comptes utilisateurs — patron et vendeurs5.1 DescriptionStocke les comptes des utilisateurs de l'application. En V1, un seul utilisateur existe (le patron). En V2, plusieurs vendeurs peuvent être créés avec des droits différenciés. Cette table est la racine de l'auditabilité : toutes les actions sont tracées via created_by_user_id et, pour les actions sensibles, via la table audit_logs.5.2 Définition des colonnesColonneType SQLiteContraintesDéfautDescriptionidINTEGERPK · NOT NULL · AUTOINCREMENT—Identifiant unique auto-incrémentéshop_idINTEGERFK · NOT NULL1FK → shops.id · Boutique principale de cet utilisateurnameTEXTNOT NULL—Nom complet de l'utilisateur (ex : Kofi Agbovi)pin_hashTEXTNOT NULL—Hash bcrypt du code PIN (4-6 chiffres).
